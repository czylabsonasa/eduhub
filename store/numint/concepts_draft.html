intro

we want to compute the value of the integral:

\int_{a}^{b} f(t)\text{d}t

where f:[a,b]\to {\mathbb R}.

when do we need to approximate integrals? If

  • it cannot be expressed in terms of elementary functions (e^{-x^2} for example).
  • the antiderivative (primitive function) is known, but too complex.
  • we should calculate a large number of “similar” type integrals.
  • the value of f is known only at a few points.

the approach

we are searching the approximation of \int_{a}^{b} f(t)\text{d}t in form of

I=\sum_{i=1}^n w_k f(t_k)

  • where t_1,\ldots ,t_n are the nodes (t_k\in [a,b]),
  • and w_1,\ldots ,w_n are the weights (they don’t depent on f).
  • I is called a quadrature rule (formula), its free parameters are n, t_1,\ldots ,t_n, w_1,\ldots ,w_n

interpolational quadrature formulas

  • more specifically: for a given f and nodes a=t_1,\ldots ,t_n=b and values f(t_1),\ldots,f(t_n) we determine the Lagrange interpolational polyinomial P and its exact integral value will be used to approximate \int_{a}^{b} f(t)\text{d}t.

    \begin{gather*} f(t) \approx P(t)\\ \int_{a}^{b} f(t)\text{d}t \approx \int_{a}^{b} P(t)\text{d}t \end{gather*}

  • we will learn what are the weights by using 1,2,3 nodes and how the corresponding quadrature rules are behaving.

midpoint (or tangent) method

  • one node interpolation
Importantsimple rule
  • it is exact for poly’s of degree at most 1.

I_{M,1}=(b-a)f\left(\frac{a+b}{2}\right)

  • to prove it just examine the plot
  • I_{M,1} means that we are “dividing” the interval [a,b] into 1 subintervals…
Noneplot

Noneerror

\left\vert \int_{a}^{b} f\text{d}t - I_{M,1}\right\vert \le \frac{M_2}{24}(b-a)^3

  • some details can be found here
Importantcompound rule
  • n+1 nodes, n intervals, mid-points
  • a=t_1<t_2<\ldots<t_{n+1}=b
  • the formula is for equally spaced nodes: t_k=a+(k-1)h with h=\frac{b-a}{n}
  • the mid-points: m_k=\frac{t_k+t_{k+1}}{2} for k=1,\ldots,n
  • I_{M,n}: using n subintervals

I_{M,n}=h\left[f(m_1)+f(m_2)+\ldots+f(m_{n})\right]

Noneerror
  • n+1 nodes, h=\frac{b-a}{n}

\left\vert \int_{a}^{b} f\text{d}t - I_{M,n}\right\vert \le (b-a)\frac{M_2}{24}h^2

  • add together the small-interval errors
Nonematlab 

% using n nodes compute the approx integral 
% of sin(t^2) on [a,b] w/ the mid-point method
clear;
fun=@(t) sin(t.^2);
a=0;
b=1;
n=ceil(3.2^2); % 3.2 sqrt(10)
t=linspace(a,b,n);
h=(b-a)/(n-1);
m=(t(1:n-1)+t(2:n))/2;
I=h*sum(fun(m))
E=abs(integral(fun,a,b)-I)

trapesoidal method

  • two nodes interpolation
  • it is exact for poly’s of degree at most 1.
Importantsimple rule

I_{T,1}=(b-a)\frac{f(a)+f(b)}{2}

  • to prove it just examine the plot
Noneplot

Noneerror

\left\vert \int_{a}^{b} f\text{d}t - I_{T,1}\right\vert \le \frac{M_2}{12}(b-a)^3

  • some details can be found here
Importantcompound rule
  • n+1 nodes, n (sub-)intervals
  • a=t_1<t_2<\ldots<t_{n+1}=b
  • the formula is for equally spaced nodes: t_k=a+(k-1)h with h=\frac{b-a}{n}

I_{T,n}=\frac{h}{2}\left[f(t_1)+2f(t_2)+\ldots+2f(t_{n})+f(t_{n+1})\right]

Noneerror
  • n equally sized intervals, h=\frac{b-a}{n}

\left\vert \int_{a}^{b} f\text{d}t - I_{T,n}\right\vert \le (b-a)\frac{M_2}{12}h^2

  • add together the small-interval errors
Nonematlab 

% using n nodes compute the approx integral 
% of sin(t^2) on [a,b] w/ the trapesoidal method

clc; clear;
fun=@(t) sin(t.^2);
a=0;
b=1;
for k=1:5
  n=ceil(3.2^k); % 3.2 sqrt(10)
  t=linspace(a,b,n);
  h=(b-a)/(n-1);
  I=h/2*sum([fun(t(1)),2*fun(t(2:n-1)),fun(t(n))]);
  E=abs(integral(fun,a,b)-I)
end

Simpson method

  • three nodes interpolation
  • it is exact for poly’s of degree at most 3. (do you see the jump?)
Importantsimple rule

I_{S,1}=(b-a)\frac{f(a)+4f(\frac{a+b}{2})+f(b)}{6}

  • compute the P(t) interpolational polynomial for the nodes a,\frac{a+b}{2},b and values f(a),f\left(\frac{a+b}{2}\right),f(b)
  • evaluate the integral \int_{a}^{b} P(t)\text{d}t

Importantcompound rule
  • n intervals
  • a=t_1<t_2<\ldots<t_{n+1}=b (the mid-points are taken “automatically”)
  • the formula is for equally spaced nodes: t_k=a+(k-1)h with h=\frac{b-a}{n}

I_{S,n}=\frac{1}{3}I_{T,n}+\frac{2}{3}I_{M,n}

Noneerror
  • n nodes, h=\frac{b-a}{n}

\left\vert \int_{a}^{b} f\text{d}t - I_{S,n}\right\vert \le (b-a)\frac{M_4}{90\cdot 2^5}h^4

  • by adding together the small-interval errors
Nonematlab 

% using n nodes compute the approx integral 
% of sin(t^2) on [a,b] w/ the Simpson method

clc; clear;
fun=@(t) sin(t.^2);
a=0;
b=1;
for k=2:5
  n=ceil(1.8^k); % 1.8 > 10^(1/4)
  t=linspace(a,b,n);
  h=(b-a)/(n-1);
  T=h/2*sum([fun(t(1)),2*fun(t(2:n-1)),fun(t(n))]);
  M=h*sum(fun((t(1:n-1)+t(2:n))/2));
  I=1/3*T+2/3*M;
  E=abs(integral(fun,a,b)-I)
end

a probabilistic method

  • Monte-carlo integration
  • \xi_1,\ldots,\xi_n \sim \xi \sim U(a,b) iid variables
  • \mathbb{E}(f(\xi_k))=\int_{a}^{b} f(t)\frac{1}{b-a}\text{d}t
  • I_{MC,n}=(b-a)\frac{\sum_{k=1}^n f(\xi_k)}{n}
  • \mathbb{E}(I_{MC,n})=\int_{a}^{b} f(t)\text{d}t=I
  • Chebysev inequality:

\begin{gather*} \mathbb{P}(|I_{MC,n}-I|\ge\delta)\le \frac{\mathbb{D}^2(I_{MC,n})}{\delta^2}\\ \mathbb{P}\left(|I_{MC,n}-I|\ge \frac{b-a}{n^{\frac{\alpha}{2}}}\right)\le \frac{\mathbb{D}^2(f(\xi))}{n^{1-\alpha}} \ \ \ (0<\alpha<1)\\ \mathbb{P}\left(|I_{MC,n}-I|\le \frac{b-a}{n^{\frac{\alpha}{2}}}\right)\ge 1-\frac{\mathbb{D}^2(f(\xi))}{n^{1-\alpha}} \ \ \ (0<\alpha<1)\\ \end{gather*}

  • by increasing n the chance that the estimator will be far from the the mean becomes smaller and smaller (it tends to 0 as n\to \infty)

% approx integral % of sin(t^2) on [a,b] w/ the monte-carlo method

clc; clear;
rng(22);

fun=@(t) sin(t.^2);
a=0;
b=1;
trueI=integral(fun,a,b);

n=1000;
xi=a+(b-a)*rand(1,n);
X=fun(xi);
Y=(b-a)*cumsum(X)./(1:n)-trueI;
plot(1:n,Y)
axis([1,n,-0.5,0.5])
title("error of $\int_0^2 \sin(t^2)dt$ by monte-carlo","Interpreter","latex")
saveas(gca(),"mcINT.png","png");

adaptive approach

  • the error formulas provide a theoretical guarantee for the convergence, but in practical software it is not so easy to get the required n for the required precision.
  • a naive approach can be:
    • choose a quadrature method (midpoint,trapesoid,Simpson…)
    • choose an increasing sequence n_1<n_2<ldots of integers (the numbers of subintervals)
    • compute I_{method,n_k} for k=1,...
    • stop if I_{method,n_{k+1}} and I_{method,n_{k}} are close to each other in some sense - no significant improvement by using larger number of intervals.
Nonenaive “adaptive”

% naive: trapesoidal + interval doubling (uniform)
% (large amount of unnecessary computations)
% with additional effort it can be done in a way
% that only N fun evaluations are preformed where
% N is n at the final state.


clear; clc; clf;
hold on;


fun=@(x) 22 + (x+1).^2 .* cos((2*x+1)./(x-4.3));
a=1.5;
b=3.6;
tol=0.5*1e-3;


trueI=integral(fun,a,b);

n=1;
I=(b-a)/2*(fun(a)+fun(b));
for k=1:20
  I0=I;
  n=2*n;
  t=linspace(a,b,n+1);
  I=(b-a)/(2*n)*(fun(a)+2*sum(fun(t(2:n)))+fun(b));
  if abs(I-I0)<tol*(1+abs(I)), break; end
end

Nonerecursively “adaptive”

Nonematlab 

% recursive: trapesoidal
% (large amount of unnecessary computations again)
% 

clear; clc; clf;
hold on;


fun=@(x) 22 + (x+1).^2 .* cos((2*x+1)./(x-4.3));
a=1.5;
b=3.6;
tol=1e-3;


function [I,t]=adapt(f,A,B,TOL)
  t=[A,B];
  function I1=work(a,b)
    m=(a+b)/2;
    I0=(b-a)/2*(f(a)+f(b));
    I1=(b-a)/4*(f(a)+f(m))+(b-a)/4*(f(m)+f(b));
    if ~(abs(I0-I1)<(1+abs(I1))*TOL)
      t=[t,m];
      I1=work(a,m)+work(m,b);
    end
  end
  I=work(A,B);
end





[I,t]=adapt(fun,a,b,tol);

trueI=integral(fun,a,b);


abs(I-trueI)

t=sort(t);
%tt=t(end:-1:1);
%fill([t,tt],[zeros(size(t)),fun(tt)],[0.91,0.92,0.91],"LineWidth",2);
for x=t
  plot([x,x],[0,fun(x)],"Color",[0.9,0.9,0.9]);
end

xx=linspace(a,b,400);
plot(xx,fun(xx),"Color",[0,0,1]);
title(sprintf("(recursive)\n error: %.4f | funeval: %d",abs(I-trueI),length(t)));
