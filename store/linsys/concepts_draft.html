intro

  • setting up an appropriate SLE is the hardest part of the problem solving…but here we only deal with the “how to solve it” part.

  • our aim is to solve systems of linear equations (SLE’s)

  • we have \(m\) equations \(n\) variables, a cofficient matrix \(A\) and a right hand side \(b\) \[ \begin{gather*} a_{11}x_{1}+\ldots+a_{1n}x_{n} = b_{1}\\ a_{21}x_{1}+\ldots+a_{2n}x_{n} = b_{2}\\ \vdots{}\\ a_{m1}x_{1}+\ldots+a_{mn}x_{n} = b_{m}\\ \text{or}\\ Ax=b\\ \text{or}\\ x_{1}a_{.1}+x_{2}a_{.2}+\ldots+x_{n}a_{.n}=b\\ \end{gather*} \]

  • here by \(a_{.j}\) we mean the \(j\)-th column of \(A\)

  • the third form is the most expressive one

    • it tells us that we are working in vector spaces
    • it tells us that we are searching for a linear combination of the column vectors that is \(b\) and the solution for the system (if any) is coefficiant set of the combination.
    • from linear algebra we know that we have three cases:
      • we cannot express \(b\) by means of the given system (0 solutions)
      • we can express \(b\) uniquely (1-solution)
      • we can express \(b\) in several different ways (\(\infty\)-solutions)

transformations

  • the aim is to reduce the original system to a simpler but equivalent one - which can be solved trivially

  • we denote the \(i\)-th row by \(a_{i.}\)

  • these are the equivalent (they keep the solution system intact) transformations of a LSE:

    • multiply the \(i\)-th row by a nonzero scalar: \(a_{i.}=\lambda a_{i.}, \ \ b_{i}=\lambda b_{i}\)
    • elimination: \(a_{j.}=a_{j.}-\lambda a_{i.},\ \ b_{j}=b_{j}-\lambda b_{i}\)
      • the \(lambda\) is called an eliminating factor
    • swap the \(i\)-th and \(j\)-th row: \(a_{i.}\leftrightarrow a_{j.},\ b_{i}\leftrightarrow b_{j}\)

  • each of the transformation above can be described as a matrix operation (\(M_{k}\)): \[ \begin{gather*} Ax=b\ \rightarrow \\ M_{1}Ax=M_{1}b\ \rightarrow \\ M_{2}M_{1}Ax=M_{2}M_{1}b \ \rightarrow \\ \ldots \\ M_{K}\ldots M_{2}M_{1}Ax=M_{K}\ldots M_{2}M_{1}b \end{gather*} \]

  • where each of the \(M_{k}\)’s are invertible \(m\times m\) matrices. That’s why the set of solutions does not change and every linear relationship between the column vectors will be preserved.

  • for example the elimination matrix is \(E-\lambda E_{ij}\) where \(E\) is the identity matrix, \(E_{ij}\) is a matrix of full of zeros but its \(ij\)-th element is 1.

  • at the end of these steps (hopefully) we will have a simpler system.

  • performing these steps to reduce the system is called gaussian-elimination

  • the parametric solution system is the following \[ x=\left[\begin{array}{c} \frac{6}{5}\\ -\frac{11}{20}\\ \frac{3}{10}\\ 0\\ \end{array}\right]- \lambda\left[\begin{array}{c} 1\\ -\frac{3}{2}\\ 2\\ -1\\ \end{array}\right] \]

  • we often have some restrictions for the solution, for example we need a solution with positive components. for this example we can easily fulfill this restriction.

  • this parametric solution can also be obtained by calling null(A,"rational").

complexity

  • for square matrices (\(n\times n\))
  • upper triangular form (UT): each of the elements strictly below the main diagonal is zero.
  • getting to the UT-form using Gaussian-elimination needs \({\cal O}(n^3)\) operations.
  • solving a linear system with triangular coefficent matrix needs \({\cal O}(n^2)\) operations.
    • it is the so-called back subtitution, which is for an UT-form matrix is the following:
function x=bsu(U,b)
   [n,~]=size(U);
   x=b;
   for k=(n:-1:1)
      s=dot(U(k,k+1:n),x(k+1:n));
      x(k)=(x(k)-s)/U(k,k);
   end
end

LU factorization

  • the elimination stops at UT form
  • for some classes of \(A\) square coefficient matrices we do not need swap operations to get a UT form.
  • the steps needed to reach the UT form is a sequence of invertible \(M_{k}\) matrices. It can be proved that their product is a lower triangular matrix with 1’s in its main diagonal: \[ A=LU \]
  • \(U\) is the residual UT-form matrix.
  • for example a diagonally dominant matrix always has \(LU\) factorization
  • why do we need factorizations?
  • often the right hand side is not known, but we have a model (\(A\)) to deal with the future \(b\)’s
  • this way we prepare the matrix by computing the \(LU\) form with \({\cal O}(n^3)\) operations and using this factorization for each of the future \(b\)’s we can obtain the solution with \({\cal O}(n^2)\) operations.

matlab

  • the function call [L,U,P]=lu(A) will create a \(PLU\) factorization for the parameter matrix. Almost always row-swaps will be performed.
  • the backslash (\) operator is a built-in for solving linear systems.

PLU factorization

  • for an arbitrary square matrix \(A\) it is always exist. The used row swaps are represented by the matrix \(P\), which is a permutation matrix: it is obtained by reordering the rows/columns of the identity matrix.

applications

  • properties of the original coefficient matrix using the properties of the residual \(U\) matrix

determinant

  • assuming that no row multiplications were used, just multiply together the diagonal elements of \(U\) and take its negative if odd number of row swaps were used.

rank

  • that is: the dimension of the subspace generated by the column-vectors of \(A\)
  • count the number of non-zero elements in the main diagonal of \(U\)
